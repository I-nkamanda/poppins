# POP.PINS 화면 설계서 (Screen Spec)

> 목적: POP.PINS 웹 서비스(자기주도 학습용 PBL 생성/관리 서비스)의 핵심 UX 플로우와 화면 구조를 정리한 문서  
> 대상: 프론트엔드 개발 (React + Vite + TypeScript + Tailwind, React Router 기준)

---

## 0. 공통 디자인 시스템 (요약)

### 0-1. 타이포그래피

- 폰트: `Helvetica Now` (대체: `system-ui, -apple-system, BlinkMacSystemFont, "Helvetica Neue", Arial, sans-serif`)
- 텍스트 스타일 (대략 Tailwind 매핑 예시)
  - Text 3XL : 18pt, LH 22 → `text-2xl font-bold`
  - Text 2XL : 17pt, LH 24 → `text-xl font-semibold`
  - Text XL  : 16pt, LH 16 → `text-lg font-semibold`
  - Text L   : 15pt, LH 18 → `text-base`
  - Text M   : 13pt, LH 16 → `text-sm`
  - Text S   : 12pt, LH 14 → `text-xs`
  - Text XS  : 11pt, LH 13 → `text-[11px]`

### 0-2. 컬러 (개념적 정의)

- Greys (100–900): 배경, 보더, 비활성 텍스트 등에 사용
- Accent / Primary:
  - 버튼, 진행률 바, 강조 텍스트에 사용 (PDF 상 오렌지/네이비 톤)
- 상태 컬러:
  - Success: 채점 결과의 정답 표시용 (연한 초록 + 진한 초록 텍스트)
  - Error: 오답 표시용 (연한 레드 + 진한 레드 텍스트)

### 0-3. 공통 컴포넌트

- 버튼 (Variants)
  - `Primary` : 가득 채워진 버튼, CTA용
  - `Secondary` : 아웃라인 or 배경 옅은 색
  - `Ghost` : 텍스트 버튼
  - Size: `lg`, `md`, `sm` (폼/카드 내 사용)

- 인풋
  - 기본 텍스트 인풋 (label + placeholder + helper text 가능)
  - 텍스트에리어 (자동 높이 or 고정 높이)
  - 셀렉트 박스 (드롭다운)
  - 토글 스위치 (on/off)

- 아이콘
  - 24px, 20px, 16px 세트
  - 주 사용: 진행 상태 아이콘(재생/일시정지), 체크, 경고, 설정, 화살표 등

- 카드
  - 학습 카드, 챕터 카드, 퀴즈 카드, 대시보드 위젯 등에서 동일한 라운드/쉐도우 스타일

## 2. 라우트 구조(Sitemap)

> 제안 라우트 구조 (React Router 기준)

- `/` : 랜딩 페이지 (LandingPage)
- `/login` : 로그인 페이지 (LoginPage)
- `/signup` : 회원가입 페이지 (SignupPage)
- `/dashboard` : 내 학습 대시보드 (DashboardPage)
- `/courses/new` : 새로운 교육과정 생성 (NewCoursePage)
- `/courses/:courseId` : 과정 개요/챕터 목록 (CourseOverviewPage)
- `/courses/:courseId/plan` : 학습 계획 만들기 (StudyPlanPage)
- `/courses/:courseId/sessions` : 챕터 세션 목록 (CourseSessionsPage)
- `/courses/:courseId/sessions/:sessionId` : 오늘의 학습(개념/실습/형성평가) (StudySessionPage)
- `/courses/:courseId/quiz` : 퀴즈 풀이 페이지 (QuizPage)
- `/courses/:courseId/quiz/result` : 채점 결과 페이지 (QuizResultPage)
- `/settings` : 개인 설정 (SettingsPage)
- `/todo` 또는 `/progress` : 투두리스트/진도 관리 (ToDoProgressPage) – 우선순위는 낮을 수 있음

## 3. 개별 화면 설계

### 3-1. 랜딩 페이지 (`/`)

**목적**  
서비스 소개 및 회원가입/로그인으로 유도.

**레이아웃**

1. 헤더
   - 로고 `POP.PINS`
   - 메뉴: `기능`, `가격`, `FAQ`, `로그인` (텍스트)
   - CTA 버튼: `무료로 시작하기` (Primary)

2. 히어로 섹션
   - 좌측 텍스트:
     - 헤드라인:  
       `벡터와 같은 주제를 학습할 학습자를 위한 POP.PINS` (실제 카피는 변동 가능)
     - 서브텍스트:  
       AI 기반으로 학습 계획, 개념 정리, 실습, 퀴즈까지 자동 생성해주는 서비스 설명
     - 버튼:
       - `지금 시작하기` (Primary)
       - `더 알아보기` (Secondary)
   - 우측: 서비스 화면 모형 이미지 (대시보드 캡쳐 등)

3. 기능 섹션
   - 카드 3~4개:
     - `문제의식`
     - `해결 방법`
     - `핵심 기능` (학습 계획 생성 / RAG 기반 개념 정리 / 실습 문제 / 퀴즈 생성 등)
     - `누가 사용하나요?` (대학생, 직장인, 부트캠프 수강생 등)

4. 기술 스택/라인업 섹션
   - 예: `Google Gemini`, `FastAPI`, `React`, `RAG` 등 로고 나열

5. FAQ 섹션
   - 아코디언 형태 Q/A 목록

6. 푸터
   - 서비스명, 카피라이트, 간단한 링크 (이용약관, 개인정보 처리방침 등)

### 3-2. 로그인 페이지 (`/login`)

**목적**  
기존 사용자 로그인.

**구성 요소**

- 제목: `로그인`
- 입력 필드:
  - 이메일
  - 비밀번호
- 옵션:
  - `로그인 유지하기` 체크박스
  - `비밀번호를 잊으셨나요?` 링크
- 버튼:
  - `로그인` (Primary)
  - 하단 텍스트: `계정이 없으신가요?` → `회원가입` 링크 버튼 (`/signup`)

**상태**

- [성공] 로그인 후 `/dashboard`로 이동 (PDF 상 "로그인 성공" 화면 언급)
- [실패] 오류 메시지 영역 (빨간 텍스트/보더)

### 3-3. 회원가입 페이지 (`/signup`)

**목적**  
신규 사용자 계정 생성.

**구성 요소**

- 제목: `회원가입`
- 입력 필드:
  - 이름
  - 이메일
  - 비밀번호
  - 비밀번호 확인
- 버튼:
  - `회원가입` (Primary)
- 하단:
  - `이미 계정이 있으신가요? 로그인` 링크

**가입 완료 화면**

- 간단한 완료 메시지: `가입이 완료되었습니다`
- 버튼:
  - `대시보드로 이동` → `/dashboard`

### 3-4. 내 학습 대시보드 (`/dashboard`)

PDF 20, 39페이지 참고.  

**목적**  
전체 학습 현황 요약 및 진행 중 교육과정 관리, 새 과정 생성.

**상단 헤더**

- 타이틀: `내 학습 대시보드`
- 서브텍스트: `진행 중인 모든 학습을 한눈에 관리하세요.`

**섹션 구성**

1. 상단 KPI 카드 영역 (4개)
   - `진행 중` : 현재 진행 중인 교육과정 개수
   - `완료` : 완료한 과정 수
   - `학습 시간(누적)` : 총 학습 시간 (예: `3h`)
   - `완료 비율` : 전체 과정 대비 완료된 비율 (예: `0%`)

2. 진행 중 학습 리스트
   - 각 카드 항목:
     - 상단: 과정 레벨 / 기간 / 남은 일수 (예: `beginner · 2025-10-22 ~ 2025-10-29 · D-7`)
     - 제목: `벡터 기초 1주 완성` 등
     - 진행률 바 + 퍼센트 (예: `25%`)
     - 상태: `진행 중`, `시작 전`, `완료` 등
     - 액션 버튼:
       - `계속 학습`
       - `계획 수정`
       - `삭제` or `더보기` 메뉴

3. 캐러셀 형태의 `위드 목표달성도` 또는 `최근 활동` (PDF 39페이지 하단 영역)
   - 날짜별 학습 활동 카드 슬라이드

4. 하단 `새로운 교육과정 생성` 영역
   - 큰 그레이 박스 + `+ 새로운 교육과정 생성` 텍스트/아이콘
   - 클릭 시 `/courses/new`로 이동

### 3-5. 새로운 교육과정 생성 / 학습 계획 만들기

#### 3-5-1. 학습 계획 만들기 (`/courses/:courseId/plan` 또는 `/courses/new` 단계 1)

PDF 27페이지 참고.

**목적**  
과정 생성 시, 제목/학습 목표/기간/참고 링크 등을 입력해 AI 기반 커리큘럼 생성 요청.

**레이아웃**

- 타이틀: `학습 계획 만들기`  
  서브텍스트: `학습 계획을 만들어보세요.`

- 좌측 폼 (기본 정보)
  - `제목` (예: `벡터 개념 정리 2주 완성`)
  - `학습 목표/주제` (예: `벡터 기초 개념과 연산 이해`)
  - `시작일` (DatePicker)
  - `종료일` (DatePicker)
  - `최대 학습 시간` (Select: `하루 30분`, `하루 1시간`, `하루 2시간` 등)

- 우측 폼 (현재 상태/요청사항/참고자료)
  - `현재 학습 상태` (textarea)
    - 예: `벡터 개념은 어슷하게 인트로만 들어본 상태이며…`
  - `요청 사항` (textarea)
    - 예: `하루 30분 분량으로 정리해 주세요.`
  - `학습 참고 링크` (textarea)
    - 예: 블로그/강의/교재 링크

- 하단 버튼
  - `생성` (Primary) → AI 커리큘럼 생성 API 호출 → 과정 전체 목차 페이지로 이동/업데이트

### 3-6. 과정 전체 목차 / 챕터 목록 페이지 (`/courses/:courseId`)

PDF 28, 29페이지 참고.

**목적**  
한 과정의 전체 챕터 구조 확인 및 각 챕터 세션으로 진입.

**레이아웃**

- 상단:
  - Course Title (예: `Vector Basics`)
  - Course Description (짧은 요약)

- 챕터 카드 리스트
  - 각 카드:
    - `Chapter 1`
    - `Chapter Description`
    - 진행률: `67%` 등 (우측 상단)
    - 버튼: `이어하기` (Primary / small)
  - 카드 스타일: 라운드 + 그림자, 호버 시 강조

### 3-7. 오늘의 학습 – 세션 진행 페이지 (`/courses/:courseId/sessions/:sessionId`)

PDF 30페이지 참고. (제목: `오늘의 학습 · 확률` 예시)

**목적**  
하루 단위 세션에서 `개념 정리`, `실습 과제`, `형성 평가`를 순서대로 진행.

**레이아웃**

- 상단:
  - 타이틀: `오늘의 학습 · [주제명]`
  - 서브텍스트: `전체 단계 진행: 0/3` (또는 `1/3`, `2/3`)

- 좌측: 단계 카드 리스트
  1. `개념 정리`
     - 요약 설명 (예: `벡터의 개념과 기본 연산 개념 정리`)
     - 참고 링크: `https://example.com/concept` (링크 텍스트)
     - 상태 뱃지: `완료`, `진행 중`, `대기` 등
  2. `실습 과제`
     - 설명 (예: `프로그래머스 유사 연습 문제 풀이`)
     - 참고 링크
  3. `형성 평가`
     - 설명
     - 참고 링크

- 우측: 진행 상태/네비 영역
  - `진행 상태`
    - Step 1: 개념 정리
    - Step 2: 실습 과제
    - Step 3: 형성 평가
  - 버튼: `다음 단계로 이동`  
    현재 진행 중 세션의 다음 컴포넌트로 라우팅

### 3-8. 강의/콘텐츠 뷰 (`/courses/:courseId/content/:contentId`)

PDF 31~33페이지에 `강의`, `문제`, `정답 입력/채점` 흐름이 존재.

**목적**  
개념 정리 콘텐츠(노트/동영상/외부 링크)와 문제 풀이를 같은 페이지/탭 구조로 제공.

**구성 (안)**

- 상단 탭
  - `강의` : 개념 정리 텍스트/임베디드 뷰어
  - `문제` : 객관식/주관식 문제 목록
  - `정답 입력/채점` : 풀이 입력 + 채점 버튼

- `강의` 탭
  - 마크다운 렌더링 영역 (AI가 생성한 concept md)
  - 스크롤 가능

- `문제` 탭
  - 문제 카드 리스트
    - 문제 번호, 문제 내용, 보기(객관식일 경우)
    - 사용자 입력 필드 (주관식일 경우 텍스트 입력)

- `정답 입력/채점` 탭
  - 사용자 답안을 일괄 확인
  - 버튼: `제출` → `/quiz/result` 로 이동 혹은 모달 표출

### 3-9. 퀴즈 페이지 (`/courses/:courseId/quiz`)

PDF 34페이지 참고.

**레이아웃**

- 타이틀: `퀴즈`
- 서브텍스트: `문제를 보고 답을 작성해주세요.`

- 문제 카드 리스트
  - 각 카드:
    - 문제 번호 (1, 2, 3…)
    - 질문 텍스트
    - 답변 입력 텍스트에리어 (placeholder: `자유롭게 서술하세요.`)

- 하단:
  - 추가 문제 버튼: `문항 추가` (선택 사항)
  - 큰 텍스트에리어(선택): 전체 요약/메모 입력
  - 오른쪽 하단 `제출하기` 버튼 (Primary)  
    → 채점 API 호출 후 `/quiz/result`로 이동

### 3-10. 채점 결과 페이지 (`/courses/:courseId/quiz/result`)

PDF 36페이지 참고.

**레이아웃**

1. 상단 요약
   - 큰 원형 배지에 점수 (예: `67점`)
   - 하단: `2/3 문항 정답`

2. 채점 결과 리스트
   - 각 문항별 카드:
     - 문제 번호 + 질문
     - 상태 뱃지: `정답`, `오답`
     - 배경:
       - 정답 카드: 연한 초록색
       - 오답 카드: 연한 빨간색
     - 영역:
       - `내 답안`
       - `정답`
       - `해설` (옵션)
   - 하단: `다음 세션으로`, `다시 풀기` 버튼

### 3-11. 개인 설정 페이지 (`/settings`)

PDF 35페이지 참고.

**내용 예시**

- 프로필 정보
  - 이름, 이메일, 프로필 이미지
