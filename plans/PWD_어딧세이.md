# Product Requirements Document (PWD)
## 어딧세이 (가제)
### "어떤 주제든, AI가 실습형 PBL 학습지로 만들어줍니다."

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
**어딧세이 (가제)**

### 1.2 프로젝트 비전
AI와 함께 설계하는 나만의 배움 여정을 제공하는 플랫폼

### 1.3 핵심 컨셉
- **PBL (문제기반 학습)**: 코디세이/42에서 지향하는 교육 방식의 연장 (프로그래밍/CS 너머로 확장)
- **AI**: 생성형 AI를 활용한 맞춤형 학습 콘텐츠 생성
- **PBL + AI 조합**: 두 가지를 결합한 혁신적인 학습 소프트웨어

### 1.4 프로젝트 목표
- 학습자가 원하는 주제를 입력하면 AI가 실습형 PBL 학습지로 자동 변환
- 개인 맞춤형 학습 경로 제공
- 교육 접근성 제고 및 학습 격차 해소

## 2. 사용자 페르소나

### 2.1 수진 (대학생, 23세)
**상황**: "짧은 시간, 필요한 내용을 학습"
- 비전공자(생명과)로 AI 공부 중
- 확률과 통계 기초가 잘 생각나지 않음
- 중간고사 전 3일 정도 빠르게 핵심 개념을 짚고 넘어가고 싶음

**요구사항**: 
- 빠른 시간 내 핵심 개념 학습
- 비전공자도 이해할 수 있는 수준의 설명

### 2.2 민수 (직장인, 31세)
**상황**: "빠른 기간 안에 팀 단위 리스킬링"
- 갑자기 레거시 유지보수 업무 배정 (Delphi 시스템)
- 팀원 중 Delphi 경험자 없음
- 다음 달까지 팀 교육자료 제작 필요

**요구사항**:
- 팀 단위 학습 자료 생성
- 실무에 바로 적용 가능한 실습 중심 콘텐츠

### 2.3 윤지 (교사, 35세)
**상황**: "공교육 내 실습형 AI 콘텐츠 설계"
- 도덕교사, 방과후 선생님 담당
- AI 윤리 교육 과정을 한 학기 동안 진행해야 함
- 기존 교재 부족, 대부분 영어 자료

**요구사항**:
- 한 학기 분량의 체계적인 커리큘럼
- 한국어로 된 실습형 교육 자료

## 3. 사용자 시나리오

### 3.1 학습 내용 입력 & 구체화 단계
1. 학습자가 학습을 원하는 주제를 입력
2. Gemini가 채팅을 통해 학습 내용 구체화
   - 학습 목표 명확화
   - 사전 지식 확인
   - 학습 범위 조정

### 3.2 학습 기간 및 강도 결정 단계
1. 학습자가 학습을 원하는 기간/강도를 입력
   - 예: 3일 집중, 2주 완주, 1학기 등
2. Gemini가 간단한 커리큘럼 리스트 제공
   - 단계별 학습 계획
   - 예상 소요 시간

### 3.3 PBL 교육자료 출력 및 학습
1. 1, 2단계 정보를 바탕으로 문제들을 단계별로 출력
   - 개념 정리
   - 실습 과제
   - 형성평가
2. 학습 보조를 위한 자료들도 골라줌
   - 유튜브 영상 추천
   - 참고 자료 링크

### 3.4 간단한 학습 관리 및 평가
1. 진도 체크하기 용의 형성평가 (동료평가 대체)
2. 학습 진도율 및 달성율을 볼 수 있는 간단한 대시보드
   - 완료한 챕터 수
   - 학습 시간 추적
   - 평가 점수

## 4. 기능 요구사항

### 4.1 핵심 기능

#### 4.1.1 학습 주제 입력 및 구체화
- **설명**: 사용자가 학습하고 싶은 주제를 입력하고, AI와의 대화를 통해 학습 내용을 구체화
- **입력**: 
  - 주제명 (예: "Flask 기반 CRUD 기초")
  - 난이도 선택
  - 목표 시간
  - 학습자 레벨
- **출력**: 구체화된 학습 목표 및 범위

#### 4.1.2 AI 기반 PBL 생성 (API 활용)
- **설명**: Google Gemini 2.5 Flash를 활용하여 PBL 형식의 학습 자료 자동 생성
- **입력**: 구체화된 학습 주제, 기간, 난이도
- **출력**: 
  - JSON/Markdown 형식의 학습 자료
  - 단계별 챕터 구성
  - 각 챕터별 개념 정리, 실습 과제, 평가 문제

#### 4.1.3 결과 시각화 (카드 UI 등)
- **설명**: 생성된 학습 자료를 직관적인 카드 UI로 표시
- **기능**:
  - 챕터별 카드 형태로 표시
  - 진행 상황 시각화
  - 단계별 네비게이션

#### 4.1.4 재생성 기능 (Cache 활용)
- **설명**: 동일 주제 재요청 시 캐시된 학습지 활용 또는 재생성 옵션 제공
- **기능**:
  - "Generate Again" 버튼으로 재생성 가능
  - 과거 생성한 학습/평가 문제 등 불러오기

#### 4.1.5 간단한 LMS & 대시보드
- **설명**: 학습 진도 관리 및 시각화 (투두리스트 급, 급할 경우 Notion으로 갈음)
- **기능**:
  - 학습 진도율 표시
  - 완료한 챕터 추적
  - 평가 점수 기록
  - 학습 시간 추적

### 4.2 확장 사용자 시나리오

#### 4.2.1 국내: 교육 접근성 제고
**상황**: 
- 중학교 2학년 준호, 강릉 인근 농어촌 마을 거주
- 집에서 학교까지 버스로 40분
- 학교에 과학 동아리, 코딩 수업 없음
- 유튜브로 코딩 학습 시도하나 체계적이지 않아 포기

**결과**:
- 교사 없이도 실습 → 결과 기록 → 다음 단계로 이어지는 루프 작동
- 마을 도서관 담당자나 지역 센터 직원이 "AI 보조 교사" 역할 가능

**가치 포인트**: AI가 지역간 학습 격차를 해소, 동일 학습 경험 제공

#### 4.2.2 해외: 극한 상황에서의 교육 수요 해소
**상황**: 
- 요르단의 자타리 난민 캠프
- 교사 아메드에게 아이들을 위한 교재 없음
- 어딧세이에 입력: "아랍어로 과학 기초를 가르치고 싶어요. 실험도구는 없어요."

**결과**: 
- AI가 현지 언어와 자원 수준에 맞춘 PBL 수업안을 자동 설계

**가치 포인트**: "AI가 교사를 대체하는 게 아니라, 교사가 없는 곳에 교사를 만들어준다."

## 6. 기술 스택

### 6.1 Frontend
- **프레임워크**: React.js / Vite
- **주요 기능**:
  - 학습 주제 입력 UI
  - AI 채팅 인터페이스
  - 결과 시각화 (카드 UI)
  - 대시보드

### 6.2 Backend
- **프레임워크**: FastAPI
- **데이터베이스**: MySQL / PostgreSQL (SQLAlchemy ORM)
- **주요 기능**:
  - 학습 주제, 기간, 생성 자료 저장
  - 학습자 context 관리
  - 생성된 커리큘럼, 개별 챕터 저장

### 6.3 AI Engine
- **LLM**: Google Gemini 2.5 Flash
- **워크플로우**: N8n + Gemini
- **구조**: 삼단구조 (Input Layer → AI Generator Layer → Output Layer)
- **주요 기능**:
  - Prompt 구조화
  - PBL 형식 학습 자료 생성
  - JSON/Markdown 형식 출력

### 6.4 시스템 구조
```
Input Layer (Prompt / UI)
    ↓
AI Generator Layer (LLM 백엔드)
    ↓
Output Layer (Web UI)
```

**세부 흐름**:
1. 사용자가 주제를 입력 (ex. "FLASK 기반 CRUD 기초")
2. Option: 난이도 / 목표 시간 / 학습자 레벨
3. Prompt 구조화: Context + Input → AI 노드에 던져주기 (웹훅)
4. Output format: JSON / Markdown
5. React.js / Vite 기반 viewer로 표출
