# PopPins II - Web vs Standalone 비교

## 개요

PopPins II는 두 가지 버전으로 제공됩니다:
1. **Web 버전**: 브라우저 기반 웹 애플리케이션
2. **Standalone 버전**: Tauri 기반데스크탑 애플리케이션

이 문서는 두 버전의 차이점과 각각의 장단점을 설명합니다.

---

## 주요 차이점 요약

| 항목 | Web 버전 | Standalone 버전 |
|------|---------|----------------|
| **실행 방식** | 웹 브라우저 필요 | 독립 실행 파일 |
| **설치** | 불필요 (URL 접속) | 1회 설치 필요 |
| **인터넷 연결** | 필수 | AI API만 필요 (로컬 DB) |
| **크기** | ~2MB (프론트엔드) | ~50MB (전체 포함) |
| **업데이트** | 자동 (서버 배포) | 수동 또는 자동 업데이트 |
| **데이터 저장** | 서버 DB | 로컬 SQLite |
| **성능** | 네트워크 의존적 | 로컬 처리로 빠름 |
| **보안** | 서버 관리 필요 | 로컬 데이터 보호 |
| **플랫폼** | 모든 OS (브라우저) | Windows, macOS, Linux |

**특징**:
- 클라이언트-서버 분리
- 중앙 집중식 데이터 관리

```
┌──────────────────────┐
│   Tauri Desktop App  │
│  ┌────────────────┐  │
│  │   WebView      │  │

│  └────────┬───────┘  │
│           │ IPC      │
│  ┌────────▼───────┐  │
│  │  Tauri Core    │  │

│  └────────┬───────┘  │
└───────────┼──────────┘
            │ Spawns

│  Python FastAPI      │

```

**특징**:
- 모든 컴포넌트 로컬 실행
- 단일 사용자
- 오프라인 동작 가능

## 기능 비교

### ✅ 공통 기능

두 버전 모두 동일하게 지원:

- AI 기반 학습 자료 생성
- 학습 목표 선택 (기초/실무/심화)
- 커리큘럼 자동 생성
- 개념 정리, 실습 과제 생성
- 객관식 퀴즈 (5문제)
- 고급 학습 (주관식 3문제, AI 채점)
- 대시보드 (학습 관리)
- 챕터 다운로드 (Markdown)
- RAG 기반 콘텐츠 생성

### Web 버전 전용 기능

- **다중 사용자 지원**
  - 사용자 계정 관리
  - 권한 제어
  - 공유 기능

- **클라우드 동기화**
  - 여러 기기에서 접근
  - 자동 백업

- **협업 기능** (향후)
  - 팀 단위 학습
  - 결과 공유

### Standalone 버전 전용 기능

- **완전 로컬 실행**
  - 인터넷 없이 사용 (AI API만 필요)
  - 빠른 응답 속도

- **개인 정보 보호**
  - 데이터가 로컬에만 저장
  - 서버 전송 없음

- **오프라인 캐싱**
  - 생성된 콘텐츠 로컬 저장
  - 즉시 재접근 가능

## 사용 시나리오

### Web 버전이 적합한 경우

#### ✅ 1. 여러 기기에서 접근
**시나리오**: 회사 PC, 집 PC, 태블릿에서 동일한 학습 자료에 접근하고 싶을 때

**예시**:
```
회사에서 파이썬 코스 생성 → 집에서 이어서 학습 → 태블릿에서 복습
```

#### ✅ 2. 팀 단위 학습
**시나리오**: 팀원들과 학습 자료를 공유하고 관리할 때

**예시**:
```
팀 리더가 커리큘럼 생성 → 팀원들에게 공유 → 각자 진행 상황 확인
```

#### ✅ 3. 관리자 모니터링
**시나리오**: 교육 담당자가 학습자들의 진도를 추적할 때

**예시**:
```
학생들이 각자 학습 → 선생님이 대시보드에서 전체 진도 확인
```

### Standalone 버전이 적합한 경우

#### ✅ 1. 개인 학습
**시나리오**: 혼자서 집중적으로 학습하고 싶을 때

**예시**:
```
자신의 속도로 학습 → 모든 데이터 로컬 저장 → 빠른 접근
```

#### ✅ 2. 데이터 보안 중요
**시나리오**: 회사 내부 교육 자료로 외부 서버에 데이터를 보내고 싶지 않을 때

**예시**:
```
회사 기밀 주제로 학습 자료 생성 → 모든 데이터 로컬에만 저장
```

#### ✅ 3. 불안정한 네트워크
**시나리오**: 인터넷 연결이 불안정한 환경에서 사용할 때

**예시**:
```
기차, 비행기 등 이동 중 → 오프라인으로 기존 콘텐츠 학습
```

#### ✅ 4. 빠른 성능 필요
**시나리오**: 네트워크 지연 없이 즉각적인 반응이 필요할 때

**예시**:
```
여러 챕터를 빠르게 탐색 → 로컬 DB에서 즉시 로드
```

## 장단점 비교

### Web 버전

#### 장점
- ✅ 설치 불필요 (URL만 있으면 즉시 사용)
- ✅ 어떤 기기에서나 접근 가능
- ✅ 자동 업데이트 (서버 배포로 즉시 적용)
- ✅ 데이터 백업 및 복구 용이
- ✅ 협업 기능 지원 가능
- ✅ 중앙 관리 및 모니터링

#### 단점
- ❌ 인터넷 연결 필수
- ❌ 서버 비용 발생
- ❌ 네트워크 지연 (속도 저하 가능)
- ❌ 서버 장애 시 사용 불가
- ❌ 데이터 프라이버시 우려 (서버 저장)

### Standalone 버전

#### 장점
- ✅ 완전 로컬 실행 (빠른 속도)
- ✅ 데이터 프라이버시 (로컬 저장)
- ✅ 오프라인 사용 가능 (AI API만 온라인)
- ✅ 서버 비용 없음
- ✅ 네트워크 장애 영향 최소화
- ✅ 커스터마이징 용이

#### 단점
- ❌ 설치 필요 (~50MB)
- ❌ 플랫폼별 빌드 필요 (Windows/Mac/Linux)
- ❌ 수동 업데이트 또는 자동 업데이트 구현 필요
- ❌ 여러 기기 간 동기화 어려움
- ❌ 협업 기능 제한적
- ❌ 백업 수동 관리 필요

## 성능 비교

### 응답 시간

| 작업 | Web 버전 | Standalone 버전 |
|------|---------|----------------|
| **앱 시작** | 즉시 (브라우저 열기) | ~3-5초 (앱 부팅) |
| **코스 생성** | ~15-30초 (네트워크 포함) | ~10-20초 (로컬 처리) |
| **챕터 로드** | ~1-3초 (서버 쿼리) | ~0.1-0.5초 (로컬 DB) |
| **퀴즈 채점** | ~5-10초 (API 호출) | ~5-10초 (API 호출)* |

*둘 다 Gemini API를 호출하므로 동일

### 리소스 사용

| 항목 | Web 버전 | Standalone 버전 |
|------|---------|----------------|
| **메모리** | ~100MB (브라우저 탭) | ~150-200MB (전체 앱) |
| **디스크** | ~5MB (캐시) | ~200MB (앱 + DB) |
| **네트워크** | 지속적 사용 | AI API만 사용 |

## 데이터 관리

### Web 버전

**저장 위치**: 서버 데이터베이스

**백업**: 자동 (서버 관리자)

**이동**: URL만 있으면 어디서나 접근

**공유**: 링크 공유 또는 권한 부여

### Standalone 버전

**저장 위치**: 
- Windows: `C:\Users\{username}\AppData\Roaming\PopPins II\`
- macOS: `~/Library/Application Support/PopPins II/`
- Linux: `~/.config/PopPins II/`

**백업**: 수동 (폴더 복사)

**이동**: DB 파일 복사 또는 클라우드 동기화 (수동)

**공유**: DB 파일 직접 전달 또는 재생성

---

## 보안 및 프라이버시

### Web 버전

- API 키: 서버에 저장
- 학습 데이터: 서버 DB에 저장
- 통신: HTTPS 암호화
- 위험: 서버 해킹 시 데이터 노출 가능

### Standalone 버전

- API 키: 로컬 `.env` 파일 또는 암호화 저장
- 학습 데이터: 로컬 SQLite DB
- 통신: Gemini API만 HTTPS
- 위험: 로컬 PC 해킹 시 데이터 노출 (일반적으로 낮음)

## 선택 가이드

### Web 버전을 선택하세요 🌐

- 여러 기기에서 학습하고 싶다
- 팀원들과 자료를 공유하고 싶다
- 설치 없이 바로 사용하고 싶다
- 자동 백업이 필요하다
- 관리자가 학습자를 모니터링해야 한다

### Standalone 버전을 선택하세요 💻

- 개인적으로만 사용한다
- 데이터 프라이버시가 중요하다
- 인터넷 연결이 불안정하다
- 빠른 성능이 필요하다
- 회사 내부 교육 자료를 다룬다
- 오프라인 환경에서 자주 사용한다

## 마이그레이션

### Web → Standalone

1. Web에서 모든 코스를 Markdown으로 다운로드
2. Standalone 앱 설치
3. Standalone에서 동일한 주제로 재생성
4. (향후) 데이터 내보내기/가져오기 기능 지원 예정

### Standalone → Web

1. Standalone DB에서 데이터 추출 (SQL)
2. Web 서버 DB로 임포트
3. (향후) 원클릭 마이그레이션 도구 제공 예정

## 결론

**Web 버전**과 **Standalone 버전**은 각각의 장점이 있습니다. 사용 환경과 요구사항에 따라 적절한 버전을 선택하세요.

- **협업 및 접근성**: Web 버전
- **프라이버시 및 성능**: Standalone 버전

**궁극적으로 두 버전 모두 동일한 핵심 기능을 제공하며, 선택은 사용자의 워크플로우에 달려 있습니다.**
