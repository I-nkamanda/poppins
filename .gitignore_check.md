# .env 파일 Git 보호 체크리스트

## ✅ 확인 사항

### 1. .gitignore 규칙 확인
```bash
git check-ignore -v app/.env
```
이 명령어가 `app/.env`를 반환하면 무시되고 있는 것입니다.

### 2. Git 추적 상태 확인
```bash
git ls-files app/.env
```
이 명령어가 아무것도 반환하지 않으면 Git에 추적되지 않는 것입니다.

### 3. 무시된 파일 확인
```bash
git status --ignored | grep .env
```
무시된 .env 파일들을 확인할 수 있습니다.

## ⚠️ 만약 이미 Git에 올라간 경우

### 1. Git에서 제거 (로컬 파일은 유지)
```bash
git rm --cached app/.env
git commit -m "Remove .env file from Git tracking"
git push
```

### 2. Git 히스토리에서 완전히 제거 (주의: 강력한 조치)
```bash
# BFG Repo-Cleaner 사용 (권장)
# 또는 git filter-branch 사용
```

### 3. API 키 교체

## 🔒 예방 조치

1. **커밋 전 확인**
   ```bash
   git status
   ```
   `.env` 파일이 목록에 나타나면 커밋하지 마세요!

2. **pre-commit 훅 설정** (선택사항)
   `.git/hooks/pre-commit` 파일을 만들어서 .env 파일 커밋을 방지할 수 있습니다.

3. **GitHub/GitLab 설정**
   - Repository Settings에서 Secrets로 API 키 관리
   - Actions/CI에서 환경 변수로 설정

## 📝 .env.example 사용

실제 `.env` 파일 대신 `.env.example` 파일을 사용하여:
- 필요한 환경 변수 목록을 문서화
- Git에 안전하게 커밋 가능
- 새 개발자가 쉽게 설정 가능
